text
あなたはフルスタック開発者です。以下の仕様に基づいて、
React + Node.js（Vercel Functions相当）のWebアプリを Replit 上で作成してください。

【プロジェクト名】
Claude SEO Article Generator

【機能概要】
ユーザーが入力したテキスト（お題または下書き）をもとに、
Claude API（Sonnet 4.5）で自然な日本語のSEO記事を生成し、
Replitサーバーに自動保存して、過去記事を管理できるアプリ。

---

## 【フロントエンド仕様】React

### 画面構成
1. **ヘッダー**
   - アプリ名「Claude SEO Article Generator」
   - 副題「AIが人間らしく記事を書く」

2. **メインエリア**
   - **入力フィールド**
     - テキストエリア（1つ）
     - プレースホルダ：「お題またはラフな下書きを入力してください...」
     - 最小高さ：100px、最大高さ：300px（スクロール可能）
   
   - **コントロールパネル**
     - 文字数スライダー：「目標文字数」 500-2000文字（デフォルト800）
     - スライダー上に現在値を表示
     - 生成ボタン：「記事を生成する」（青色、目立つ）
     - キャンセルボタン：（生成中のみ表示）
   
   - **生成中の表示**
     - 「記事を生成中...」とテキスト表示
     - ローディングアニメーション（スピナーまたはドット）
     - 進捗を示す（例：「Claude APIと通信中...」）
   
   - **結果エリア**
     - 完成した記事をテキストエリアに表示
     - 記事の下に：
       - 「コピー」ボタン（クリップボードにコピー）
       - 「ダウンロード」ボタン（テキストファイルで保存）
       - 「新規作成」ボタン（フォームをリセット）
   
   - **サイドバー（左側）**
     - 「過去の記事」というタイトル
     - 生成済み記事の一覧（最新順）
     - 各記事：
       - タイトル（入力テキストの最初の30文字）
       - 生成日時
       - 文字数
       - クリックで本文を右側に表示
       - 削除ボタン（🗑️）

### デザイン
- モダンで落ち着いた色合い（濃紺 + 白 + アクセントは緑）
- レスポンシブ（スマホ対応）
- フォント：sans-serif

## 【バックエンド仕様】Node.js（Vercel Functions）

### API エンドポイント

#### 1. POST `/api/generate-article`
**リクエスト**
```json
{
  "prompt": "ユーザーが入力したテキスト",
  "targetLength": 800
}
処理

Claude API（claude-sonnet-4-5-20250929）を呼び出し

temperature: 0.9

max_tokens: 2000

システムプロンプト：

text
あなたはSEOに精通した、人間味あふれるベテランWEBライターです。
ユーザーの入力（お題または下書き）をもとに、{targetLength}文字前後の記事を作成してください。

【執筆スタイル】
・自然な日本語（〜だよ、〜だね）で、親しみやすさを重視
・AIっぽさを消すため、適度に砕けた表現や筆者の感情を15%混ぜてください
・適度に改行を入れ、スマホで読みやすい構成に
レスポンス

json
{
  "article": "生成された記事のテキスト",
  "characterCount": 850,
  "generatedAt": "2026-01-25T19:39:00Z",
  "id": "記事の一意なID"
}
2. GET /api/articles
機能

保存済み記事の一覧を返す（最新順）

レスポンス

json
{
  "articles": [
    {
      "id": "unique-id-1",
      "title": "入力テキストの最初の30文字...",
      "content": "記事の全文",
      "characterCount": 850,
      "generatedAt": "2026-01-25T19:39:00Z"
    }
  ]
}
3. GET /api/articles/:id
機能

指定IDの記事を取得

4. DELETE /api/articles/:id
機能

指定IDの記事を削除

【データベース】
SQLiteを使用（Replitのファイルシステム内）

テーブル: articles
text
- id (TEXT, PRIMARY KEY)
- title (TEXT)
- content (TEXT)
- characterCount (INTEGER)
- inputPrompt (TEXT)  // 元のユーザー入力（後で参考になるかも）
- generatedAt (DATETIME)
【自動保存機能】
記事生成直後、自動的にサーバーに保存

ユーザーは保存ボタンを押す必要なし

保存成功時：画面に「✓ 記事を保存しました」とトースト表示（2秒後に消える）

【環境変数】
CLAUDE_API_KEY: Anthropic APIキー
（Replitの「Secrets」に設定済みを想定）

フロントエンド: React 18 + React Router + Axios

バックエンド: Node.js + Express

データベース: SQLite3

ホスティング: Replit（開発環境）→ Vercelデプロイ予定
